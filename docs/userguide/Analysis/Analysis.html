Analysis#

In the root MPCD repository, we provide a series of scripts to help visualise simulations output by MPCD. These scripts are written in Python, and require Python 3, and required packages are included (defectHandler.py, shendrukGroupFormat.py).

Important

  • The shendrukGroupFormat.py script is not included in the repository, but is available on the Shendruk-Lab GitHub page. It requires a matplotlib style sheet to be installed. Follow the readme instructions to install it.

  • If you choose not to use the shendrukGroupFormat.py script, you will need to comment out the call that imports it and modify rare lines that use it.

Important

  • Many of the scripts require ffmpeg to be installed on your system to produce movies. If you do not have ffmpeg installed, you can still run the scripts, but they will not produce movies. You can install ffmpeg using your system’s package manager, or by downloading it from the FFmpeg website.

In this section, we briefly describe their usage. Examples of their usage can be found in the tutorials section.

Note

These analysis scripts are provided primarily to quickly verify simulation output for first time users, and to give an example of how to parse MPCD’s output files. Users are expected to develop their own analysis scripts appropriate for their own purposes.

Common#

Most scripts make use of Python’s argparse library to parse command line arguments. Arguments have tried to be kept consistent between scripts, despite some differences in their usage. Here, we briefly describe some common features

Help#

In general, you can always give the --help (shorthand -h) flag to any script to see a list of arguments and their usage.

For example:

$ python3 analysisScripts/orientationFieldAnimator.py -h

gives output

usage: orientationFieldAnimator.py [-h] [--qx QX] [--qy QY] [-c LENGTH] [-a MYASPECT] [-k KEEPFRAMES] [-d DEFECTDATA] dataname inputname start finish avdim

Orientation field rendering script.

positional arguments:
dataname              Path to the data (should be directorfield.dat)
inputname             Path to input .json file
start                 Starting timestep for averaging
finish                Finishing timestep for averaging
avdim                 Dimension to 'slice' over

optional arguments:
-h, --help            show this help message and exit
--qx QX               Only show every qx arrow in x
--qy QY               Only show every qy arrow in y
-c LENGTH, --length LENGTH
                      Length of director lines
-a MYASPECT, --myAspect MYASPECT
                      'auto' or 'equal'
-k KEEPFRAMES, --keepFrames KEEPFRAMES
                      0=don't keep (delete) frames; 1=keep frames
-d DEFECTDATA, --defectData DEFECTDATA
                      Path to defect data (if any)

The output may vary depending on script, but will always be the most up-to-date description of the script’s arguments.

Arguments preceded by – are optional.

Common arguments#

The following arguments are (generally) consistent between scripts:

  • dataname

    Path to the input data file. Usually a .dat file. The exact file will vary from script to script.

  • inputname

    Path to the input .json file corresponding to the given data. This is used to read in simulation parameters.

  • -s/ start

    Starting timestep index for plotting.

  • -f/ finish

    Ending timestep index for plotting. Set to a very large number to plot until the end of the simulation.

  • -a/ avdim

    For scripts that will “slice” over a dimension, this is the slicing dimension. These scripts will average in this dimension, hence the name avdim. Accepts "x", "y", or "z".

  • -A/ --myAspect

    (Optional, default "auto") The aspect ratio to produce plots in. Either "auto" for automatic aspect ratio (resizes axes to fit the figure size), or "equal" for equal aspect ratio (keeps the actual aspect ratio of the simulation domain).

  • -k/ --keepFrames

    (Optional, default 0) When producing frames for a movie, whether to keep the frames or delete them after the movie is produced. Set to 1 to keep the frames, and 0 to delete frames.

  • -p/ --savePDF

    (Optional, default 0) When producing frames for a movie, save the frames as PDFs to use in papers/talks. Set to 1 to save the frames as PDFs, and 0 to not save them.

  • -g/ --groupStyle

    (Optional, default 0) Use the shendrukGroupStyle Matplotlib style file for plotting. Set to 1 to use group style, and 0 to not use it.

  • -d/ --defectData

    (Optional, default None) Path to defect data file, if any. This is used by some scripts to plot defects in the field.

  • qx:

    (Optional, default 1) Only show every qx-th spatial step in the \(\hat{x}\) direction.

  • qy:

    (Optional, default 1) Only show every qy-th spatial step in the \(\hat{y}\) direction.

colourbar flag shendruk-group-style flag scale c better

densityAnimator.py#

Purpose#

The ‘densityAnimator’ Python script is designed to create a 2D animation of a scalar field that represents the number of particles in each cell.

Command to run#

python3 analysisScripts/densityAnimator.py [dataname] [inputname] -s [start] -f [finish] -a [avDim] -i [sliceIndex] -A [myAspect] -k [keepFrames] -N [normalise] -p [savePDF] -d [defectData]

Most arguments are described in the consistent arguments section, except sliceIndex and normalise.

  • sliceIndex:

    Index of the plane to be sliced for averaging. Set the number of frames to skip between. If you want all frames to be plotted set 0.

  • normalise:

    Normalise the colourbar on the right side.

Outputted files#

A video will be outputted. Individual PNG frames (named frameXXXX.png) are automatically compiled into an MP4 movie using ffmpeg. If --savePDF is set to 1, every frame of the movie will be kept as a pdf file and if --keepFrames` is set to one the ``.png frames will not be deleted.

flowFieldAnimator.py#

Purpose#

The ‘flowFieldAnimator’ Python script is designed to create a 2D animation of the vector field that represents the centre of mass velocity of the particles in each cell.

Command to run#

python3 analysisScripts/flowFieldAnimator.py [dataname] [inputname] -s [start] -f [finish] -a [avdim] --qx [QX] --qy [QY] -A [myAspect] -k [keepFrames] -p [savePDF] -d [defectData]

Most arguments are described in the consistent arguments section.

Outputted files#

A video will be outputted. Individual PNG frames (named frameXXXX.png) are automatically compiled into an MP4 movie using ffmpeg. If --savePDF is set to 1, every frame of the movie will be kept as a pdf file and if --keepFrames` is set to one the ``.png frames will not be deleted. It will also output a figure of the time-averaged velocity field as a colourmap and as a contour.

orientationFieldAnimator.py#

Purpose#

The ‘orientationFieldAnimator’ Python script is designed to create a 2D animation of the director field of a nematic liquid crystal film.

Command to run#

python3 analysisScripts/orientationFieldAnimator.py [dataname] [inputname] -s [start] -f [finish] -a [avdim] --qx [qx] -- qy [qy] -l [length] -A [myAspect] -k [keepFrames] -d [defectData]

Most arguments are described in the common section. The exceptions are as follows:

  • length:

    (Optional, default 0.5) Length of the director lines.

  • defectData:

    (Optional, default "") Path to a topological charge field data file. If unset, or set to "", then no defects will be drawn. Otherwise, attempt to load and then draw defects from file.

Outputted files#

A video of the director field coloured by the scalar order parameter will be outputted. Individual PNG frames (named frameXXXX.png) are automatically compiled into an MP4 movie using ffmpeg. If -k is set to 1, then every frame of the movie will be kept.

probDistScalar.py#

Purpose#

The ‘probDistScalar’ Python script is designed to plot probability density functions of scalar quantities. It will average the times and/or output a movie.

Command to run#

python3 analysisScripts/probDistScalar.py [dataname] -X [xaxis] -s [start] -f [finish] -a [plotAv] -m [makeMovie] -k [keepFrames]

Most arguments are described in the consistent arguments section, except

  • xaxis:

    Label for the x-axis (what scalar quantity it is a probability density of)

  • makeMovie:

    Whether or not to animate temporal data (0 or 1)

  • plotAv:

    Whether or not to plot average (0 or 1)

Outputted files#

A graph of the temporal average and/or a movie.

ensAvPlotter.py#

Purpose#

The ‘ensAvPlotter’ Python script is designed to plot ensemble averaged quantities as a function of time. It is assumed that the first column is time and the column numbers start from 0.

Command to run#

python3 analysisScripts/ensAvPlotter.py [dataname] -y [yaxis]  -c [columns]

None of these arguments are described in the consistent arguments section.

  • dataname:

    The path to the data file

  • yaxis:

    Label for the y-axis (what is being plotted)

  • columns

    A list of the columns that you would like to plot

Outputted files#

A graph will be outputted.

autocorrPlotter.py#

Purpose#

The ‘autocorrPlotter’ Python script is designed to plot the spatial autocorrelation of quantities, when the autocorrelations were already computed in the main NAIMH-MPCD simulation. It will average the times and/or output a movie.

Command to run#

python3 analysisScripts/autocorrPlotter.py [dataname] [yaxis] -s [start] -f [finish] -a [plotAv] -m [makeMovie] -k [keepFrames]

Most arguments are described in the consistent arguments section, except

  • yaxis:

    Label for the y-axis (what is being autocorrelated)

  • makeMovie:

    Whether or not to animate temporal data (0 or 1)

  • plotAv:

    Whether or not to plot average (0 or 1)

Outputted files#

A graph of the temporal average and/or individual PNG frames (named frameXXXX.png) that are automatically compileed into an MP4 movie using ffmpeg.

swimmerTraj_fieldsAnimator.py#

Purpose#

The ‘swimmerTraj_fieldsAnimator’ Python script is designed to create a 2D animation of a scalar field that represents the number of particles in each cell.

Command to run#

python3 analysisScripts/swimmerTraj_fieldsAnimator.py [dataPath] [inputname] [fieldType] -s [start] -f [finish] -a [avDim] --qx [QX] --qy [QY] -A [myAspect] -k [keepFrames] -p [savePDF] -d [defectData]

Most arguments are described in the consistent arguments section, except dataPath and fieldType

  • dataPath:

    While the other analysis scripts take a specific data file as an argument, this script takes the path where both the swimmers.dat and flowfield.dat (or directorfield.dat) results are.

  • fieldType:

    The velocity (vel) or vorticity (vor), director (dir), or no (none) fields can be plotted. Velocity and vorticity require flowfield.dat, while the director requires directorfield.dat.

Outputted files#

A video called swimmerVelField_animation.mp4, swimmerVorField_animation.mp4, swimmerDirField_animation.mp4 or swimmer_animation.mp4 will be outputted, depending on which argument was given for fieldtype. Individual PNG frames (named frameXXXX.png) are automatically compiled into an MP4 movie using ffmpeg.

swimmerRefFrame_flow.py#

Purpose#

The ‘swimmerRefFrame_flow’ Python script produces a plot of the average flow field around a dumb dumbbell swimmer. The swimmer is always centered in the final plot, and there is the possibility to rotate it so that the averaging is always done with respect to swimmer orientation. An animation can be produced of the unaveraged process.

Command to run#

python3 analysisScripts/swimmerRefFrame_flow.py [dataPath] [inputname] -s [start] -f [finish] -z [avDim] -r [rotAx] --qx [QX] --qy [QY] -A [myAspect] -k [keepFrames] -f [flowtag]

Most arguments are described in the consistent arguments section, except dataPath rotAx, and flowtag

  • dataPath:

    While the other analysis scripts take a specific data file as an argument, this script takes the path where both the swimmers.dat and flowfield.dat (or directorfield.dat) results are.

  • rotAx:

    Select wether to only center the swimmer (0) in the plot or to also rotate it so that it lies along the x axis (1).

  • flowtag:

    If using a time-averaged flow field (flowfield.dat), set to 0; If using an instantaneous velocity field (velocityfield.dat), set to 1.

Outputted files#

A pdf image called swimmerVelField_refFrameRotated_av.pdf or swimmerVelField_RefFrameCentred_av.pdf depending on the value of rotAx. If -m is set to 1, a movie called swimmerVelField_refFrameRotated_animation.mp4 or swimmerVelField_RefFrameCentred_animation.mp4 is also produced from individual PNG frames (named frameXXXX.png) that are automatically compiled into an MP4 movie using ffmpeg. If -k is set to 1, every frame of the movie will be kept as a png file.

topochargeFieldAnimator.py#

One of the key features of liquid crystals is the existence of topological defects, which are singularities in the director field. An option to get these in MPCD is to output topological charge fields, which are scalar fields that are zero everywhere except where defects are detected.

Note

  • Topological charge fields can only be dumped in 2D, and likewise this script only works in 2D.

  • NIAMH-MPCD has a built in defect identifier that analyzes the topological charge on the fly. The file-size of individual defects is much smaller than the topological charge field.

A method to visualise these fields is to use this script.

Command to run#

python3 analysisScripts/topochargeFieldAnimator.py [directorData] [topoData] [inputname] -s [start] -f [finish] -a [avdim] --qx [qx] -- qy [qy] -l [length] -d [deflength] -A [myAspect] -k [keepFrames]

Most arguments are described in the common section. The exceptions are as follows:

  • directorData:

    Path to a director field data file. Should be directorfield.dat.

  • topoData:

    Path to a topological field data file. Should be topochargefield.dat.

  • length:

    (Optional, default 0.5) Length of the director lines.

  • deflength:

    (Optional, default 1.5) Length of the lines used to draw defects. Recommended to be ~3x larger than the director lines.

Outputted files#

A video is produced. Individual PNG frames (named frameXXXX.png) are automatically compiled into an MP4 movie using ffmpeg. If -k is set to 1, then every frame of the movie will be kept.

polymer-2Dprojection.py#

Purpose#

The ‘polymer-2Dprojection’ Python script generates 2D animations of either the flow field or the nematic director field from hybrid MPCD-MD simulations for a single polymer. It overlays scalar maps (velocity magnitude or scalar order), directional arrows (velocity or director), polymer chains, and optional topological defects. The script projects 3D simulation data onto a 2D plane, based on a user-specified projection axis (\(\hat{x}\), \(\hat{y}\), or \(\hat{z}\)). It supports both 3D and 2D simulations. For 2D simulations, the system is internally padded with a third dimension, but the user must still specify a projection axis.

Note

  • This script requires a polymer VMD file (with -vmd.vtf in its name) to function properly. It will fail if the file is not present in the input directory.

Command to run#

python3 analysisScripts/polymerAnimator.py [datapath] [fieldType] -s [start] -f [finish] -a [avdim] --qx [qx] -- qy [qy] -l [length] -A [myAspect] -k [keepFrames] -p [savePDF] -d [defectData]

Most arguments are described in the consistent arguments section, except datapath and fieldType.

Arguments#

  • datapath: Path to the folder (must contain input.json, field data, and polymer VTF file).

  • fieldType – Choose between vel (if want to plot the velocity field from flowfield.dat) or dir (if want director field from directorfield.dat).

    Note

    Even in 2D simulations, a projection axis must be specified.

Outputs#

The script saves individual high-resolution PNG frames (named frameXXXX.png) and automatically compiles them into an MP4 movie using ffmpeg. The output file is named flow.mp4 or director\_defX.mp4, depending on the selected field and defect flag.

decomposedHydroMomentsTADPole.py#

Purpose#

The ‘decomposedHydroMomentsTADPole’ Python script is designed to analyze the moments of the flows generated by swimmers (ensemble and time averaged). It will produce two figures and analysed data.

Command to run#

python3 decomposedHydroMomentsTADPole.py [dataPath]

Arguments#

  • <dataPath> – Path to the folder (must contain swflowfield.dat, which is outputted from NIAMH-MPCD by adding the “swFlowOut” tag to your .json input file.

Outputs#

This will output a plot of the flow around the swimmers (FlowArountTADPole.png), the decomposed moment data files (url2.npy) and a plot of the dipolar component of the flow around TADPoles as a function of distance from the hydrodynamic centre of the TADPole (DipolarComponent.pdf).
